Quick and easy tag cloud in Pelican

By Stefaan Lippens on 2026/09/28
Tagged: pelican, trick, python

Here's a quick trick to create a tag cloud in Pelican without plugings and minimal amount of code.

Define a jinja filter in pelicanconf.py to convert a tag count to a CSS font-size value, for example like this:

import math
JINJA_FILTERS = {
    # ...
    'tag_count_to_font_size': lambda c: '%.1f%%' % (100 + 25 * math.log(c, 2)),
}

(Play with the parameters of that mathematical function to tweak the size mapping appropriately.)

In the tags.html template you can now use this filter to inject inline CSS font sizes as desired. For example, in a typical implementation that renders the tags as an item list:

 <ul class="tags">
     {%- for tag, articles in tags|sort %}
        <li style="font-size: {{ articles|count|tag_count_to_font_size }};">
            <a href="{{ SITEURL }}/{{ tag.url }}">{{ tag }}</a>

For a pretty tag cloud, you also need some CSS rules on the corresponding list elements (if not already), in essence something like this:

    ul.tags { list-style-type: none; }
    ul.tags li { display: inline-block; }

and probably some more margin and padding tweaking.
